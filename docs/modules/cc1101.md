# CC1101 — Sub-GHz Transceiver

## Role

A sub-GHz radio that can **receive and transmit** on the common ISM bands. This is the module that mirrors the Flipper Zero's sub-GHz feature: capture the signal of a remote you own, then replay it. The same chip that ships inside the Flipper.

- **Bands:** 300–348 MHz, 387–464 MHz, 779–928 MHz (315 / 433 / 868 / 915 MHz typical)
- **Interface:** SPI
- **Direction:** TX + RX

## Wiring

SPI bus — full pinout in [`../wiring.md`](../wiring.md). Summary: VCC→3.3 V, SCK→GPIO11, MOSI→GPIO10, MISO→GPIO9, CSN→GPIO8 (CE0), GDO0→GPIO25.

> Add a 100 µF + 100 nF capacitor pair across VCC/GND — the CC1101 is noise-sensitive.

## Kali setup

Enable SPI on the Pi:

```bash
sudo raspi-config        # Interface Options → SPI → Enable
# or, headless:
echo 'dtparam=spi=on' | sudo tee -a /boot/config.txt
```

Driving the CC1101 is done from Python over SPI. A common library:

```bash
sudo apt install python3-pip
pip3 install spidev
# plus a CC1101 driver library of your choice
```

## Core commands / workflow

The CC1101 has no CLI tool like `rtl_sdr`; you talk to it through SPI registers. Typical flow exposed by the NexusPi backend (`backend/modules/subghz.py`):

| Step | What happens |
|---|---|
| Configure | Set frequency, modulation (ASK/OOK or 2-FSK), data rate |
| Listen | Put the chip in RX, read incoming packets via GDO0 interrupt |
| Record | Store the captured bit pattern / timing |
| Replay | Switch to TX, re-emit the stored pattern |

## Example — capture & replay your own remote

1. Set the CC1101 to the remote's frequency (often 433.92 MHz for cheap remotes).
2. Listen and press the remote button → the backend records the OOK timing.
3. Replay the stored capture on TX to reproduce the command.

This is the canonical "clone my garage remote" demo — only on hardware you own.

## Limits & legal

- Sub-GHz **transmission is regulated.** Stick to license-free ISM bands within legal power, and only transmit toward devices you own or test in a shielded environment.
- Never capture or replay third-party devices (car key fobs, neighbours' gates, public infrastructure). See [`threat-model.md`](threat-model.md).
